## 3  Physics discussion: the p^t > 0 ergosphere check

### Question
> In the raytracer we ensure that pt > 0 for each ray.  Do we physically need to do this
> for rays within the ergosphere of a Kerr black hole?

### Answer

**Yes — p^t > 0 is required everywhere, including inside the ergosphere.**

`pt` in the code is the *contravariant* time component of the photon 4-momentum,
p^t = dt/dλ.  A photon with p^t < 0 would be moving backward in coordinate time, which is
always unphysical regardless of location.

The formal statement: p^t > 0 is equivalent to the ZAMO (Zero Angular Momentum Observer)
measuring a positive energy.  Working out the ZAMO 4-velocity
u^μ_ZAMO = (1/√e2ν, 0, 0, ω/√e2ν) and its covariant components:

```
u_{t,ZAMO} = g_{tt}·u^t + g_{tφ}·u^φ
           = (e2ν − ω²e2ψ)/√e2ν  +  ω²e2ψ/√e2ν
           = e2ν/√e2ν  =  √e2ν          (ω²e2ψ terms cancel exactly)

u_{φ,ZAMO} = g_{φt}·u^t + g_{φφ}·u^φ
           = ωe2ψ/√e2ν  −  ωe2ψ/√e2ν  =  0
```

Therefore:

    E_ZAMO = p^μ u_{μ,ZAMO} = p^t · √e2ν

Since e2ν = ρ²Δ/Σ² > 0 outside the event horizon (Δ > 0 there, ergosphere included):

    **E_ZAMO > 0  ⟺  p^t > 0**

The cancellation of the ω²e2ψ terms is algebraic and holds regardless of the sign of
g_{tt} (which changes at the ergosphere boundary).  The derivation breaks down only at the
horizon (Δ = 0, e2ν = 0), consistent with no physical observer being stationary there.

**What is special about the ergosphere** is not that p^t can be negative, but that frame
dragging (ω) is large enough that photons with strongly retrograde angular momentum h can
have their effective "frame-dragging contribution" ω·h approach their conserved energy k.
At the radius where k = ω·h the formula

    p^t = (Σ²k − 2ar·h) / (ρ²Δ)

gives p^t = 0.  This is the physical turning point — no null geodesic with real pr/pθ
exists there — so the code's `if(pt <= 0)` check correctly terminates such rays.

## 4  Is the p^t check equivalent to dotting with the Killing vector?

### Question
> Is the E_ZAMO check equivalent to dotting the photon 4-momentum with the timelike
> Killing vector?

### Answer

No.  There are three distinct objects:

| Code variable | Mathematical object | Conserved? | Can fail mid-propagation? |
|---|---|---|---|
| `k` | p_t (covariant) — dot product with ∂/∂t | Yes | No |
| `pt` | p^t (contravariant) | No | Yes |
| E_ZAMO | √e2ν · p^t | No | Yes |

The Killing vector ξ^μ = (1, 0, 0, 0) gives:

    g_{μν} ξ^μ p^ν = g_{tν} p^ν = p_t = k

This is the conserved covariant component, stored as `k` in the code.  Since k is constant
along the geodesic and positive by initialisation, checking k > 0 can never catch an
ergosphere crossing mid-propagation.

`pt` in the code is the *contravariant* p^t, computed at each step from k, h via
`momentum_from_consts`.  E_ZAMO = √e2ν · p^t uses p^t, not p_t.  The existing `pt <= 0`
check is already the ZAMO future-direction condition; there is no additional check to add.

### Validity inside the ergosphere

All of the above holds inside the ergosphere (r_horizon < r < r_ergosphere) because:

- e2ν = ρ²Δ/Σ² > 0 (Δ > 0 there)
- The algebraic cancellation giving u_{t,ZAMO} = √e2ν and u_{φ,ZAMO} = 0 depends only
  on the metric structure, not on the sign of g_{tt}
- The chain E_ZAMO > 0 ⟺ p^t > 0 is unbroken throughout the exterior region

The code's `if(pt <= 0)` check (Euler propagator line ~248, RK4 propagator lines ~870
and ~1100) is already the physically correct future-direction condition.
