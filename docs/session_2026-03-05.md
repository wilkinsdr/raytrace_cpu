## 2  `RayDestination::step_limit()` design and implementation

### Design

Added a virtual `step_limit(r, theta, phi, pr, ptheta, pphi)` method to `RayDestination<T>`.  Given the current position and first-stage momenta, it returns the step size that would just reach the boundary under linear extrapolation.  The default returns `std::numeric_limits<T>::max()` (no cap) — the correct behaviour for geometries where no meaningful limit can be defined.

### Changes — `ray_destination.h`

- Added `#include <limits>`.
- Updated file header and base-class comment block to document `step_limit()`.
- **`RayDestination<T>` base**: added virtual `step_limit()` returning `max()`.
- **`FlatDiscDestination<T>`**: override returns `(theta_lim − theta) / ptheta` when approaching the disc (`ptheta > 0`, `theta < theta_lim`), mirrored for the reverse-tracing (`theta_lim < 0`) case; otherwise returns `max()`.
- **`DiscWithISCODestination<T>`**: identical logic but additionally returns `max()` immediately when `r < r_isco` or `r > r_out` (no point capping the step where `reached()` would return false anyway).

### Changes — `raytracer.cpp` (`propagate_rk45_dest`)

- Updated the step-cap comment block to reflect the new intent.
- Inside the `while (!step_accepted)` retry loop, immediately after the `rlim` clamp, added:

```cpp
{
    T h_dest = dest->step_limit(r, theta, phi, pr1, ptheta1, pphi1);
    if (h_dest < h_try) { h_try = h_dest; clamped = true; }
}
```

The `clamped = true` flag prevents a boundary-truncated step from updating the running adaptive step size.

## 4  Photon-ring arc diagnosis

The user observed a thin arc of pixels near the edge of the image that appeared to map to disc positions far from the main disc image.

**Setup:** `imageplane_disc_image_isco`, spin = 0.998, inclination = 85°, image plane ±5 rg, `r_disc = 15`.

**Diagnosis via Python analysis of `dat/imageplane_test.fits`:**

Vertical cut at x = 0 revealed three distinct bands of disc hits:

| Image-plane y | Disc r | Redshift g | Description |
|---|---|---|---|
| −5.0 to −3.6 | 1.24–3.2 | 0.09–0.54 | Lower lensed arc |
| −3.6 to −1.3 | — | — | Escape gap (no hits) |
| −1.3 to −0.15 | 1.3–14.9 | 0.09–0.90 | Main disc image (far side) |
| −0.15 to +3.5 | — | — | Escape gap (no hits) |
| +3.5 to +5.0 | 1.24–3.2 | 0.10–0.54 | Upper lensed arc |

**Conclusion:** The arc is the **first-order gravitationally lensed (photon-ring) image of the inner disc**.  Rays at large |β| in the image plane graze the photon sphere, orbit roughly half-way around the black hole, and land close to the ISCO on the near side of the disc.  The strong redshift (g ≈ 0.1–0.5) and small disc radius (just above the ISCO at r ≈ 1.24 for a = 0.998) are the characteristic signatures.  No anomalous (numerically scattered) pixels were found — the arc is smooth, coherent, and physically correct.

To suppress these in a scientific analysis, pixels can be separated by image-plane |β| position or by a disc-radius / redshift threshold.
